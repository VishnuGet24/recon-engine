4. Authentication and Session Flow
Auth type: Flask session-based auth (no JWT)

Implemented:

Login endpoints: /login and /auth/login
Logout endpoints: /logout and /auth/logout
Session check endpoints: /me and /auth/me
No public registration endpoint
Admin-only user creation endpoint: /admin/users
Passwords hashed with bcrypt
Identity accepted on login:

Username or email (same login form/API)
Session hardening:

Session fixation defense (session.clear() before login session set)
HttpOnly cookie enabled
SameSite configurable
Secure cookie configurable via env
5. CSRF + Rate Limiting
Implemented in security.py:

CSRF token issued and stored in session
CSRF required for state-changing methods (POST, etc.)
CSRF endpoint: /csrf and /auth/csrf
Frontend sends token via X-CSRF-Token header
Basic in-memory rate limiting:
Login attempts limited per IP
Scan submissions limited per session/IP fallback
6. RBAC and Permission Enforcement
Decorators + runtime checks enforce permissions:

@login_required
permission_required(...)
role_required(...)
Scan authorization logic:

passive requires scan:passive
active requires scan:active
full requires scan:active and admin role check in runtime flow
HTTP behavior:

Unauthorized actions return 403
Unauthenticated actions return JSON 401/redirect (based on request type)
7. Scan Execution Lifecycle
Endpoints:

POST /scan (preferred unified endpoint, accepts scan_mode)
GET /scan
GET /scan/<id>
Also available: /scans, /scans/passive, /scans/active, /scans/full
Flow:

Validate session user
Validate scan mode and permission
Insert scan row as running
Execute passive/active logic
Store full JSON into scans.result_json
Set status to completed or failed
Set timestamps
Compute/store basic risk fields (risk_score, overall_risk, confidence_score)
Write audit logs for requested/completed/failed events
